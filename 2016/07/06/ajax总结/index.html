ajax总结

php

  1. 必须写在<?php?>中

  2. 可以定义变量，变量必须以$开头

  3. php的特殊字符串，用来向客户端输出的

  4. 当文中输出中文的时候，为了解决乱码，可以加一个头文件

  5. php中有两种类型的数组

    • 普通数组

      通过array这个函数去定义。当我们想要遍历数组的时候，发现php中没有length这个属性，这个时候我们想要获取数组的长度就得使用count(数组名)来获取长度

    • 关联数组

      通过键值对的方式来存储数据，也就是说数组里面的数据有关联关系

    • 二维数组

  6. 在php中字符串的拼接使用.而不是+

  7. php中定义函数，调用函数

  8. php中常见的输出语句

    • echo 这个是用来输出字符串的
    • print_r() 用来输出数组或者对象的
    • var_dump() 这个是用来输出数组里面的详细信息

  9. php中常见的函数

    • count 统计数组的长度(前文提到过)
    • file_get_contents(文件) 读取一个文件里面的内容，读取出来的内容是字符串
    • in_array(元素，数组) 判断数组中是否存在某个元素，这个是用在普通数组当中的
    • array_key_exists(key,数组) 判断数组中是否存在某个key，这个是用在关联数组中的

  10. Refresh响应头的介绍

    但是有的浏览器不遵守这个协议，要自己写代码完成。

GET提交方式

  1. get提交方式有哪些

    • 在地址栏输入地址发送请求
    • 点击超链接也属于get提交
    • 表单提交，默认是get提交

  2. get方式提交的数据都是参数和参数值的方式，可以传递多个参数参数值，并且显示在地址栏，在第一对键值对前面用？，后续多个参数之间使用&隔开。

    • 上传的数据在目标地址中做了什么？

      1. 接收数据，接收客户端传递到服务端的数据

      2. 应为是GET方式提交，所以就要用$_GET去接收数据，接收到之后，对应的值是一个数组，是一个关联数组。

      3. 处理数据

      4. 返回数据

POST提交方式

  1. 表单提交可以改成post提交方式

  2. 后台用$_POST的方式去接收数据

  3. 关于POST提交方式的一个文件上传的小案例

    • 必须是表单提交
    • 表单里面必须有一个input标签的type=”file”
    • 必须是post提交方式
    • 必须给表单设置一个属性enctype=”multipart/form-data”

    • 后台接收表单提交的数据要用$_FILES接收

      这里上传的文件有一个临时存储的位置，如果你想要改变存储的位置可以使用move_uploaded_file(临时存储的位置，想要保存的位置)方法。

http协议

  1. http协议的数据格式分为两部分
    • 客户端发送到服务器的请求，成为请求的数据格式
    • 服务器响应给客户端的数据，成为响应的数据格式
  2. 传递的数据格式
    • 请求的数据格式(客户端发送给服务器)
      1. 请求首行：告诉服务器我要去请求的资源地址。并告诉HTTP协议的版本
      2. 请求头：请求头的名称和请求头的值
      3. 请求空行
      4. 请求体
    • 相应的数据格式(服务器响应给客户端)
      1. 响应首行：状态码
        • 200 OK
        • 404 请求的资源没有找到
        • 500 服务器内部错误
        • 405 请求的方式不支持
        • 304 没变，请求的资源没有发生变化
        • 302 重定向
      2. 响应头
      3. 响应空行
      4. 响应体
    • get和post请求的区别
      1. get 提交请求的数据都在地址栏中，相对来说不安全
      2. get 请求的数据都在请求地址后面，对请求的数据大小有限制，限制大概1kb
      3. get 没有请求体
      4. post 请求的数据在请求体当中，相对来说安全
      5. post 请求的数据大小没有限制。文件上传，这个必须是post方式提交
      6. post 提交有一个特殊的请求头：Content-Type:application/x-www-form-urlencode可以对请求的数据包含中文的数据进行编码
      7. get 比post 的请求头要少些，性能要高一些.

ajax

  1. 同步交互

    • 客户端请求服务器，服务器接收到请求，处理请求，然后给客户端一个响应。响应回来的数据会把原来的页面给覆盖。我们把这种交互称为同步交互。

  2. 异步交互

    • 客户端请求服务器，服务器接收到请求，处理请求，然后给客户端一个响应。响应回来的数据不会把原来的页面给覆盖掉。我们把这种交互称为异步交互。

    • 异步交互实际上就是页面不刷新，就可以动态的去获取服务端的数据

      原理
xml（可扩展标记语言）：xml是数据格式，因为服务器跟客户端进行交互的时候，数据量太大，为了方便解析，就规定这样的一个格式，这是一个比较通用的数据格式，基本上任何语言里面都对他有支持
XML的步骤
      1. 创建这样的一个对象
        var xhe=new XMLHttpRequest();
        1. 我要跟服务端建立一个链接
          xhr.open("GET","01ajax.php?username=chengjun");
          1. 我要在连接上面发送数据，get方式发送数据，数据在地址的后面。如果是post，就将数据写在这一步里面（这一步不能省略）
            xhr.send(null);
          2. 接收服务端返回的数据
            xhr.onreadystatechange=function(){}
            在这个对象上面注册了一个回调函数，在服务器端响应数据的时候调用，监听服务器端状态的改变
            xml语法
          3. 文档声明，必须写在第一行
          4. 有且仅有一个一个根元素
          5. 标签可以嵌套，但是不能交叉嵌套

json

  1. 它是一种轻量级的数据格式，以更少的字符串来代表更多的数据
  2. json比xml更加轻量级，目前基本都用json
  3. json的格式/表示数据
    • 以键值对的方式来表示数据 key：value
    • 表示一条数据 {“key”:value,”key”:value}
    • 表示多条数据 [{“key”:value,”key”:value}，{“key”:value,”key”:value}]
    • key 对应得值可以是任意类型
  4. json数据解析
    • js中有提供方法，将后台的json转换成对象
      • eval(): eval(“(“+json+”)”); 如果json格式有问题，转换会失败
      • JSON.parse();
    • 服务端：header(“Content-Type:text/json;charset=utf-8”)

jQuery中的ajax操作

  1. jQuery方法可以理解为两种类型

    • 全局方法（一般我要扩展的是工具方法，我就扩展全局方法）

    • 局部方法（扩展的方法要做页面上的DOM元素，就扩展局部方法）

  2. get 方法 用来发送get 请求的

  3. post 方法 用来发送post 请求的

  4. getScript 方法 用来载入一段远程的js代码

  5. getJSON 方法 发送一个请求给后台，后台返回的数据必须是json格式的数据

  6. load 方法 这个是用来载入一个页面

  7. serialize方法，单词序列化 将表单里的数据拼接成字符串
    serializeArray方法 序列化数组，将表单里面的内容组装成一个数组

  1. 作用

    页面发送请求，服务器返回数据，返回的一般是json格式的数据，返回了之后
    解析数据，解析了数据之后我们要将数据跟标签组装在一起，然后将数据放在页面上。

  2. 用法步骤

    • 导入文件

    • 创建一个模板

    • 准备模板内容，模板里的内容与页面上的数据有关系

    • 将数据(这里要求一定是个对象)跟模板进行绑定，调用template()的方法去绑定

跨域

  1. 什么是跨域：我访问a站点，得到一个网页，然后再a站点的网页想去访问b站点的资源，这个就是跨越。跨越是浏览器的一中安全限制，浏览器会禁止去访问b站点的资源。

  2. 如果域名、协议、端口都相同是可以访问的，不受限制。
    端口、域名、协议有一个不相同就跨域了。

  3. 跨域的形式

    • 第一种跨域：一些公司有很多小系统，每个小系统又都有二级域名这时相互访问也是跨域，但是可以解决

    • 第二种跨域

      • 当主域名也不相同的时候，我在这个页面下还是想用XMLHttpRequset对象去访问另外的站点的数据，这时形成了跨域。

      • 我们使用jQuery的ajax方法来解决跨域，我们为ajax设置一个属性dataType：jsonp

  4. jsonp和cors跨域有什么不同（必须理解）

    • jsonp
      1. jsonp只能用get方式提交，cors都支持
      2. 以后开发的过程中，肯定是采用jsonp多
      3. 如果就从开发的角度来讲，我们直接$.ajax({dataType:”jsonp”,})就解决了
      4. jsonp底层通过script标签发送请求，发送到服务端一个回调函数，这个回掉函数的名称通常叫做callback，然后我们在客户端定义一个全局的回掉函数，服务器返回的是一个回调函数。会将服务端返回的数据作为回调函数的参数传递到客户端
      5. 这个方式老牌的浏览器都支持jsonp 的方式。
    • cors
      1. cors在服务区段设置一个响应头：Access-Control-allow-Origin:允许哪些站点可以访问
      2. 但是一些老牌的浏览器不支持这个响应头:Access-Control-allow-Origin
      3. 如果客户端发送到服务端的参数数据量大的时候必须开启post 跨域。

  5. iframe标签

    • 作用：在当前页面引入一个新的页面

    • 可以通过获取该元素并修改引入页面的内容
