http协议

  1. http协议的数据格式分为两部分
    • 客户端发送到服务器的请求，成为请求的数据格式
    • 服务器响应给客户端的数据，成为响应的数据格式
  2. 传递的数据格式
    • 请求的数据格式(客户端发送给服务器)
      1. 请求首行：告诉服务器我要去请求的资源地址。并告诉HTTP协议的版本
      2. 请求头：请求头的名称和请求头的值
      3. 请求空行
      4. 请求体
    • 相应的数据格式(服务器响应给客户端)
      1. 响应首行：状态码
        • 200 OK
        • 404 请求的资源没有找到
        • 500 服务器内部错误
        • 405 请求的方式不支持
        • 304 没变，请求的资源没有发生变化
        • 302 重定向
      2. 响应头
      3. 响应空行
      4. 响应体
    • get和post请求的区别
      1. get 提交请求的数据都在地址栏中，相对来说不安全
      2. get 请求的数据都在请求地址后面，对请求的数据大小有限制，限制大概1kb
      3. get 没有请求体
      4. post 请求的数据在请求体当中，相对来说安全
      5. post 请求的数据大小没有限制。文件上传，这个必须是post方式提交
      6. post 提交有一个特殊的请求头：Content-Type:application/x-www-form-urlencode可以对请求的数据包含中文的数据进行编码
      7. get 比post 的请求头要少些，性能要高一些.

ajax

  1. 同步交互

    • 客户端请求服务器，服务器接收到请求，处理请求，然后给客户端一个响应。响应回来的数据会把原来的页面给覆盖。我们把这种交互称为同步交互。

  2. 异步交互

    • 客户端请求服务器，服务器接收到请求，处理请求，然后给客户端一个响应。响应回来的数据不会把原来的页面给覆盖掉。我们把这种交互称为异步交互。

    • 异步交互实际上就是页面不刷新，就可以动态的去获取服务端的数据

      原理
xml（可扩展标记语言）：xml是数据格式，因为服务器跟客户端进行交互的时候，数据量太大，为了方便解析，就规定这样的一个格式，这是一个比较通用的数据格式，基本上任何语言里面都对他有支持

跨域

  1. 什么是跨域：我访问a站点，得到一个网页，然后再a站点的网页想去访问b站点的资源，这个就是跨越。跨越是浏览器的一中安全限制，浏览器会禁止去访问b站点的资源。

  2. 如果域名、协议、端口都相同是可以访问的，不受限制。
    端口、域名、协议有一个不相同就跨域了。

  3. 跨域的形式

    • 第一种跨域：一些公司有很多小系统，每个小系统又都有二级域名这时相互访问也是跨域，但是可以解决

    • 第二种跨域

      • 当主域名也不相同的时候，我在这个页面下还是想用XMLHttpRequset对象去访问另外的站点的数据，这时形成了跨域。

      • 我们使用jQuery的ajax方法来解决跨域，我们为ajax设置一个属性dataType：jsonp

  4. jsonp和cors跨域有什么不同（必须理解）

    • jsonp
      1. jsonp只能用get方式提交，cors都支持
      2. 以后开发的过程中，肯定是采用jsonp多
      3. 如果就从开发的角度来讲，我们直接$.ajax({dataType:”jsonp”,})就解决了
      4. jsonp底层通过script标签发送请求，发送到服务端一个回调函数，这个回掉函数的名称通常叫做callback，然后我们在客户端定义一个全局的回掉函数，服务器返回的是一个回调函数。会将服务端返回的数据作为回调函数的参数传递到客户端
      5. 这个方式老牌的浏览器都支持jsonp 的方式。
    • cors
      1. cors在服务区段设置一个响应头：Access-Control-allow-Origin:允许哪些站点可以访问
      2. 但是一些老牌的浏览器不支持这个响应头:Access-Control-allow-Origin
      3. 如果客户端发送到服务端的参数数据量大的时候必须开启post 跨域。

  5. iframe标签

    • 作用：在当前页面引入一个新的页面

    • 可以通过获取该元素并修改引入页面的内容

颜色

标签（空格分隔）： CSS3

新增了RGBA、HSLA模式，其中的A 表示透明度通道，即可以设置颜色值的透明度，跟opacity相比，它们不具有继承性，即不会影响子元素的透明度。

  • RGBA

    • Red：0-255
    • Green：0-255
    • Blue：0-255
    • Alpha：0-1（小数）
    • 注意，不使用A那么认为没有透明度

  • HSLA

    • Hue(色调):0-360
    • Saturation(饱和度):0-100%
    • Lightness(亮度):0-100%
    • Alpha(透明度):0-1（小数）

  • 取色：

    • 使用PS的拾色器即可
    • 在线颜色转换

  • 透明度说明：

    • opacity只能针对整个盒子设置透明度，子盒子及内容会继承父盒子的透明度 ，他是作为CSS的属性来使用的
    • transparent始终完全透明，他是作为颜色的取值来使用color:transparent

  • 适用范围：

    • 所有可以使用颜色的地方，都能够使用RGBA 以及HSLA

DOCTYPE的作用

一个标准的HTML文档,开头第一句都应该是<!DOCTYPE>,根据HTML版本的不同,写法上会有所差别.

  • tips:
    • DOCTYPE并不是HTML标签,而是一个声明
    • 作用是:告诉浏览器按照哪个规则去解析页面

DOCTYPE写法_HTML4中

在HTML4.01中有三种DOCTYPE声明,但是随着技术的发展,这几种写法很少使用了,在一些老的网站中才可以看到


  • 1.HTML 4.01 Strict
    Emmet语法为: html:4s tab


  • 2.HTML 4.01 Transitional
    Emmet语法为: html:4t tab


  • 3.HTML 4.01 Frameset
    没有找到对应的Emmet语法

DOCTYPE写/法_HTML5中

Emmet语法为:html:5+tab或者!+tab

Emmet语法创建文档结构

  • html:5或!：HTML5文档类型
  • html:xt：XHTML过渡型文档类型
  • html:xs：XHTML严格型文档类型
  • html:4t：HTML4过渡型文档类型

  • html:4s：HTML4严格型文档类型

keygen 元素是密钥对生成器（key-pair generator）。当提交表单时，会生成两个键，一个是私钥，一个公钥。
私钥（private key）存储于客户端，公钥（public key）则被发送到服务器。公钥可用于之后验证用户的客户端证书（client certificate），即实现非对称加密
目前，浏览器对此元素的糟糕的支持度不足以使其成为一种有用的安全标准。

多媒体

早期的因特网主要用来分享学术成果,但是对普通民众而言,更愿意在上面分享一些更有趣的内容,比如视频,音频,但这些技术在html5之前都不是由html标签提供的

网页音视频解决方案发展

虽然早期的html并没有提供支持视频或者音频播放的标签,但是这并不影响人们分享的欲望

  • 支持方式1:
    • 使用embed直接将视频塞入页面,然后就可以使用Windows Media Player,Apple QuickTime或者其实的视频播放器来创建播放窗口
    • 但是这种方式对于视频本身不可控,兼容性问题也无法顾及
  • 支持方式2
    • 使用浏览器插件,一个是微软的Silverlight,还有使用最普遍的Adobe Flash
    • Flash不但完全解决了浏览器支持问题,而且装机率之高让人咋舌(基本上99%的计算机都安装了Flash播放器)
    • 使用Flash播放视频除了要学习Flash这项技术本身以外,更关键的是在iPhone,ipad并不支持这项技术
    • 如果想要查看视频的播放方式,将鼠标移动到视频窗口,右键如果可以看到Flash等文字,那么该网站使用的就是Flash插件
  • 多媒体标签:
    • Html5为了解决使用Flash的各种问题推出了多媒体标签
    • 由于视频格式问题,不同的浏览器对于相同格式的视频支持不同,需要准备多份视频
    • 无法对播放的视频提供很好的保护效果,因为用户可以直接对视频文件另存为
  • 总结:
    • 虽然html5中的多媒体标签有各种不好,但我们还是需要拥抱这项新的技术,因为他的用法,真的十分简单

H5_data*属性

data-*这个属性解决了自定义Html标签内属性取值的问题,只要我们想,可以往HTML标签中添加任意个数的属性

  • 语法:
    • data-:这个是固定格式
    • data-xxx:xxx是自定义的值
  • 注意:
    • 属性名不应该包含任何大写字母，并且在前缀 “data-“ 之后必须有至少一个字符
    • 属性值可以是任意字符串
    • 不要使用纯数字,特殊字符等作为自定义值
    • 如果data之后使用-连接多个值,取值时,需要使用驼峰命名
